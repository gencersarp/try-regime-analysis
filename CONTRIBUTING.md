# Contributing to try-regime-analysis

We welcome institutional-grade contributions to this research framework.

## Standards
- **No Look-ahead Bias**: Every change must be validated against point-in-time constraints.
- **Typed Code**: All new functions must include Type Hints.
- **Tested**: All core logic must have unit tests with >80% coverage.
- **Linted**: Use `ruff check` and `ruff format` before submitting.

## Process
1. Fork the repo.
2. Create a branch for your feature/fix.
3. Submit a PR with detailed numerical results and ablation studies.
